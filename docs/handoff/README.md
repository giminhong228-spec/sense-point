# 디자인 핸드오프

## 이 시스템이 하는 일

```
기민 (피그마에서 코드 복사) → 문서에 붙여넣기 → 성용이 코드에 반영
```

각 항목에 **파일:위치**가 적혀 있는 이유: 기존 코드(before)와 피그마 코드(after)를 비교해서 정확하게 반영하기 위함.

---

## 기민님 작업 가이드

### 작업 흐름
1. 페이지별 `.md` 파일을 연다
2. **디자인 요소명**을 보고, 피그마에서 같은 요소를 찾는다
3. 피그마에서 해당 요소의 **코드를 복사**한다
4. **피그마 값** 란에 복사한 코드를 **통째로 붙여넣는다**
5. 질문이 있으면 **메모** 란에 적는다
6. 완료되면 `- [ ]`를 `- [x]`로 체크한다

> (파일:위치에 적힌 CSS 클래스명은 성용이 코드에 적용할 때 쓰는 참고용 — 몰라도 됨)

### 피그마에서 코드 복사하는 법
| 디자인 내용 | 피그마에서 하는 법 |
|------------|------------------|
| 스타일 (색상, 폰트, 간격 등) | 요소 선택 → Inspect 패널 → CSS 복사 |
| 애니메이션/모션 | 요소 선택 → 복사형식에서 코드 복사 |
| 인터랙션 변경 | 프로토타입 패널에서 인터랙션 설정 복사 |
| 새 요소 / 구조 변경 | 해당 프레임 선택 → 코드 복사 (CSS/SVG 등) |
| SVG 아이콘 | 요소 선택 → Copy as SVG |

### 핵심 원칙

> **모든 디자인 의도는 코드로 뽑아서 피그마 값에 붙여넣는다.**
> 스타일이든, 애니메이션이든, 인터랙션이든, 구조 변경이든 — 전부 코드로.
> 코드가 들어가야 성용이 정확하게 반영할 수 있다.

**메모**는 코드가 아닌 것만: 질문, 확인 요청, 의도 보충 설명.

### 막혔을 때
- **15분만 시도**하고, 그래도 모르겠으면 바로 질문
- 질문할 때: **몇 번 항목**에서 막혔는지 + **스크린샷** 첨부
- 하단 **질문/메모** 섹션에 자유롭게 기록해도 됨
