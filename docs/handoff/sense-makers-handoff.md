작성일 : 2026.04.13  
작성자 : 기민  
전달 대상 : 성용

---

## 1. 페이지 컨셉

Sense Makers 상세 페이지는 성우 님 인터뷰 형식의 콘텐츠 페이지다.  
사용자가 글을 읽으며 자신의 감상을 메모로 남기고, 다른 사람의 담벼락을 볼 수 있는 구조.

---

## 2. 레이아웃 구조

```
Sense Makers (페이지 최상위)
  ├── Header          (Fixed — 스크롤 시 상단 고정)
  └── Body
        ├── Content   (좌측 70% — 본문 스크롤 영역)
        └── Sidebar   (우측 30% — 고정, 내부 독립 스크롤)
```

### 좌우 개별 스크롤 (미구현 — 개발 단 처리)

- 좌측 본문과 우측 사이드바는 각각 독립적으로 수직 스크롤
- 좌측 : `overflow-y: auto`
- 우측 사이드바 : `position: sticky; top: [헤더 높이]px` / 내부 메모 리스트 `overflow-y: scroll`

---

## 3. 본문 구조 (Content)

```
Content
  ├── 아티클 헤더 (히어로 이미지 / 카테고리 / 제목 / 메타)
  ├── 본문
  │     ├── 인트로
  │     ├── 본문 섹션 (반복) ← 소제목 + 본문 텍스트 + 이미지
  │     └── 에필로그
  ├── 또 다른 이야기들 (관련 콘텐츠)
  └── 담벼락
```

### 본문 이미지 배치 바리에이션 (미구현 — 개발 단 처리)

본문 각 섹션 내 이미지 배치는 다양한 변주를 줄 예정.  
시선의 재미를 위해 섹션마다 배치 방식을 달리한다.

| 케이스 | 배치 방식 |
|--------|-----------|
| Case A | 이미지 중앙 정렬 (1장) |
| Case B | 이미지 좌측 or 우측 정렬 (1장) |
| Case C | 이미지 2열 배치 (2장) |

- 이미지 하단에 캡션 텍스트 포함
- 피그마 내에선 대표 케이스만 표현. 나머지는 본 명세 참고하여 개발 단 처리.

---

## 4. 또 다른 이야기들 (관련 콘텐츠)

### 구조

- 관련 아티클 카드 3개 가로 나열
- 하단 READ MORE 버튼

### 호버 인터랙션 (미구현 — 개발 단 처리)

콘텐츠 리스트 페이지의 카드 호버 인터랙션과 동일한 방식.

- 이미지 위로 그라데이션 딤 오버레이
  - `background: linear-gradient(to top, rgba(0,0,0,0.6) 0%, rgba(0,0,0,0) 100%)`
- 상세 정보 (제목, 카테고리, 읽기시간) 아래에서 위로 슥 등장
  - `transform: translateY(12px) → translateY(0)`
  - `opacity: 0 → 1`
  - `transition: 200ms ease`
- 클릭 시 해당 아티클 페이지로 바로 진입

---

## 5. 담벼락

### 구조

- 상단 : 섹션 타이틀 "담벼락" + 필터 버튼 (친구랑 보기 / 다같이 보기)
- 담벼락 카드 목록
- 하단 : "N개의 담벼락 모두 보기" 버튼

### 담벼락 카드 타입

| 타입 | 설명 |
|------|------|
| Default | 본문 없이 메모 텍스트만 |
| Quote | 상단에 보라빛 인용 발췌 영역 + 하단 메모 텍스트 |

### 담벼락 클릭 인터랙션 (미구현 — 개발 단 처리)

- 담벼락 카드 클릭 시 우측 사이드바가 담벼락 탭으로 전환
- 사이드바 내 메모 상태 ↔ 담벼락 상태 탭 이동

---

## 6. 우측 사이드바

### 상태 구분

| 상태 | 설명 |
|------|------|
| 메모 상태 (기본) | 사용자 메모 리스트 + 하단 + 버튼 |
| 담벼락 상태 | 담벼락 카드 목록 (오버레이로 씌워짐) |

### 메모 상태 — 메모 블록 (MemoBlock)

메모 블록은 피그마 컴포넌트로 구성. 컴포넌트명 : `MemoBlock`

**Variant**

| 속성 | 값 |
|------|-----|
| Type | Free / Quote |
| State | Default / Hover |

- `Type=Free` : 사용자가 직접 텍스트 입력한 메모
- `Type=Quote` : 본문 텍스트를 드래그 후 우클릭으로 생성한 메모. 상단에 인용 발췌 영역 표시.

**MemoBlock 구조**

```
MemoBlock
  ├── top       (드래그 핸들 영역 — 배경 투명)
  │             (왼쪽 상단 모서리 접힌 삼각형 그래픽 포함)
  ├── memo      (텍스트 입력 영역 — 흰색 배경)
  └── bottom    (공개범위 토글 + 날짜 — 흰색 배경)
```

**공개범위 토글 (Cyclic Toggle)**

클릭 시 순환 전환. 드롭다운 아님.

순서 : 나만보기 → 친구랑보기 → 전체공개 → 나만보기 (무한 순환)  
각 상태별 아이콘 + 텍스트 교체. 색상 변화 없음.

**메모 추가 및 이동 기능 (미구현 — 개발 단 처리)**

- 상단 `top` 영역을 드래그하여 메모 순서 변경
- 하단 + 버튼으로 빈 메모 추가
- 본문 텍스트 드래그 → 우클릭 → Quote 타입 메모 자동 생성 (anchor 연결 포함)

### 담벼락 상태

- 메모 사이드바 위에 오버레이로 씌워짐 (Open overlay)
- 담벼락 진입 트리거 : 담벼락 카드 클릭 또는 더보기 버튼 클릭
- 복귀 : 뒤로가기 버튼 클릭 → Close overlay → 메모 상태로 복귀

| 항목 | 사유 | 처리 방법 |

| 관련 콘텐츠 호버 인터랙션 | 피그마 모션 한계 | 본 문서 명세 참고 |
